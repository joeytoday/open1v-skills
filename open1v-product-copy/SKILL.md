# 产品推广文案写作

根据产品信息生成适配不同渠道的推广文案，每次任务创建独立项目文件夹。支持公众号文案一键发布到草稿箱。

## 核心流程

```
收集产品信息
    ↓
❶ 创建项目文件夹
    ↓
❷ 按渠道生成文案（公众号 / 小红书 / 详情页）
    ↓
❸ 保存为独立 Markdown 文件
    ↓
❹ 发布公众号（可选：Markdown → 杂志风格 HTML → 微信草稿箱）
    ↓
❺ 交付确认 & 迭代
```

## 工作流程

### ❶ 收集产品信息

向用户确认以下信息（已知的跳过）：

- **产品名称**：具体产品名
- **核心卖点**：3 个以内的关键卖点
- **目标人群**：谁会买/用这个产品
- **投放渠道**：公众号 / 小红书 / 详情页（默认全选）

如用户未指定渠道，使用 ask_question 工具询问（支持多选）。

> 如果配合 `open1v-product-visual` Skill 使用，可从商品识别结果中自动获取产品信息。

### ❷ 创建项目文件夹

每次任务创建独立文件夹，三个渠道的文案各一个 Markdown 文件：

```bash
mkdir -p ./copy-<商品短名>-<MMDD>/
```

**命名规则**：
- `<商品短名>`：2-4 个中文字或英文词，如 `充电宝`、`earbuds`
- `<MMDD>`：当天日期
- 示例：`./copy-充电宝-0610/`

**目录结构**：

```
copy-充电宝-0610/
├── mp.md          ← 公众号文案（800-1500 字，故事性深度长文）
├── xhs.md         ← 小红书文案（300-500 字，种草短文）
└── detail.md      ← 详情页文案（结构化卖点模块）
```

### ❸ 按渠道生成文案

#### 公众号文案 → `mp.md`

**风格**：专业深度 + 故事性，800-1500 字

**结构**：
```
标题（价值导向，15 字以内）
↓
场景引入（读者痛点共鸣，2-3 句）
↓
产品介绍（核心卖点展开，每个卖点配使用场景）
↓
使用效果/数据（可量化的价值）
↓
行动号召（引导关注/购买）
```

**要求**：
- 开头用场景/故事吸引，不要硬广开场
- 适当使用 emoji 增强可读性
- 段落短小，适配手机阅读
- 注意开头 3 句的吸引力，这决定了打开率

#### 小红书文案 → `xhs.md`

**风格**：口语化 + 种草感，300-500 字

**结构**：
```
标题（含 emoji，制造好奇，20 字以内）
↓
开头（第一人称体验分享，1-2 句吸引）
↓
使用体验（按卖点分段，每段 2-3 句）
↓
总结推荐（真诚表达，不要太硬）
↓
标签（5-8 个相关话题标签）
```

**要求**：
- 第一人称"我"的口吻
- 口语化，像在跟朋友聊天
- 多用感叹号和 emoji
- 每段不超过 3 行
- 避免"最""第一""绝对"等绝对化用语
- 结尾附话题标签，如 `#好物推荐 #提效神器`

#### 详情页文案 → `detail.md`

**风格**：结构化 + 卖点驱动，按模块输出

**结构**：
```
主标题 + 副标题
↓
核心卖点模块（每个卖点：标题 + 一句话描述 + 场景图描述）
↓
产品参数/规格
↓
使用场景展示（3-5 个场景）
↓
用户评价/社会证明（如有）
↓
购买引导
```

**要求**：
- 每个模块独立，方便设计师拆解排版
- 卖点描述简短有力，不超过 2 句
- 场景图描述可直接作为 `open1v-product-visual` 的生图 prompt 使用

### ❹ 发布公众号文章到草稿箱（可选）

当用户说"发布到公众号"或提供了 appId/appSecret 时，触发此流程。

#### Step A：检测环境 & 自动配置

首先自动检测依赖和配置是否就绪，**不足的部分直接执行安装/配置，不要求用户手动操作**：

```bash
# 1. 检查 npm 依赖是否已安装，没有则自动安装
cd <skill目录>/open1v-product-copy
[ -d node_modules ] || npm install

# 2. 检查公众号配置是否存在
# 按优先级查找：.mp-config.json → 环境变量 → Obsidian 插件配置
```

**如果未找到任何配置**，使用 ask_question 工具向用户索要 appId 和 appSecret：

```
请提供你的微信公众号 appId 和 appSecret：
（在 微信公众平台 → 设置与开发 → 基本配置 中获取）
```

拿到后**立即执行 init 保存**（用户以后不用再提供）：

```bash
node scripts/publish.mjs init --app-id <用户提供的appId> --app-secret <用户提供的appSecret>
```

#### Step B：检查 IP 白名单

自动获取当前出口 IP 并提示用户确认是否已添加白名单：

```bash
curl -s ifconfig.me
```

向用户展示 IP 并询问："这个 IP 是否已加入公众号白名单？（微信公众平台 → 设置与开发 → 基本配置 → IP白名单）"

- 用户说"已加" → 继续
- 用户说"没加" → 告知用户去添加，等用户确认后继续

> IP 白名单是微信平台限制，必须由用户在公众号后台手动添加，无法通过 API 自动完成。这是整个流程中**唯一需要用户额外操作**的步骤。

#### Step C：自动发布

所有准备就绪后，直接执行发布：

```bash
cd <skill目录>/open1v-product-copy
node scripts/publish.mjs publish <项目文件夹>/mp.md --cover <封面图路径>
```

脚本自动完成：
1. 获取 access_token（自动缓存 2 小时）
2. 上传封面图到微信素材库
3. 将 `mp.md` 转换为内联样式的 HTML（使用内置杂志风格 CSS）
4. 上传正文中的本地图片到微信图床
5. 创建草稿 → 告知用户"已发布到草稿箱，请在公众号后台查看"

**封面图来源**（按优先级）：
1. 用户指定的图片路径
2. 配合 `open1v-product-visual` 生成的商品图
3. 使用 `open1v-mpcover-gen` Skill 生成封面（如已安装）
4. 向用户索要

**内置杂志风格**（`assets/product-post.css`）：
- 留白充裕、字体优雅、排版克制
- 品牌红（#c0392b）强调色 + 暖灰配色体系
- h1 居中大标题 / h2 底线栏目标题 / h3 红色竖线段落引导
- 引用块杂志 pull-quote 风格
- 圆角卡片式代码块和表格斑马纹
- 优雅三点分割线

> **Markdown 样式转化发布参考**：本发布功能的 Markdown → HTML 转换逻辑参考了 [obsidian-mp-publisher](https://github.com/joeytoday/obsidian-mp-publisher) 项目。

### ❺ 交付确认 & 迭代

展示项目文件夹内容和各渠道文案摘要，询问用户是否满意。

**快速迭代**：

| 问题 | 操作 |
|---|---|
| 调整某个渠道的语气 | 直接修改对应 md 文件 |
| 增加新渠道版本 | 在文件夹中新建 md 文件 |
| 公众号文案不满意 | 修改 mp.md 后重新 `publish` |
| 换公众号样式 | 编辑 assets/product-post.css 后重新发布 |

## 注意事项

- 不同渠道的文案独立输出到各自的 md 文件，不要混在一起
- 小红书文案避免绝对化用语
- 公众号文案注意开头 3 句的吸引力
- 详情页文案的场景图描述可配合 `open1v-product-visual` 生成配图
- 如用户提供了竞品信息，可做差异化对比（但避免贬低竞品）
- 发布公众号时，正文中的图片必须上传到微信图床，外链图片会被拦截

## 环境准备（Agent 自动执行，用户无需手动操作）

在执行任何步骤前，Agent 自动检测并处理：

```bash
# 检查 npm 依赖，未安装则自动安装
cd <skill目录>/open1v-product-copy
[ -d node_modules ] || npm install
```

### 公众号发布配置（首次发布时自动引导）

Agent 自动按以下优先级查找配置，**全部找不到时用 ask_question 向用户索要并自动保存**：

1. **已保存的本地配置**：`.mp-config.json`（init 命令生成）
2. **环境变量**：`MP_APP_ID` / `MP_APP_SECRET`
3. **命令行参数**：`--app-id` / `--app-secret`（临时使用）
4. **Obsidian mp-publisher 插件**：自动读取（兼容已有用户）

> 用户只需在首次发布时提供一次 appId 和 appSecret，Agent 自动保存后续无需再提供。

## 目录结构

```
open1v-product-copy/
├── SKILL.md
├── package.json                ← Node 依赖（marked + juice）
├── .mp-config.json             ← 公众号配置（init 命令生成，.gitignore）
├── assets/
│   └── product-post.css        ← 杂志风格公众号排版 CSS
└── scripts/
    └── publish.mjs             ← 公众号发布脚本（init/token/convert/draft/publish）
```
